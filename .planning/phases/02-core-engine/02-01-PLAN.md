    Create `src/lib/wordValidator.ts` implementing:

    1. **Internal helpers** (not exported):
       - `expandQTiles(word: string): string` — `word.toUpperCase().replaceAll('Q', 'QU')`. Used ONLY for dictionary lookup. Never stored in game state.
       - `letterFrequency(word: string): Map<string, number>` — frequency counter for multiset operations. Uses `word.toUpperCase()`.
       - `isMultisetSuperset(newWord: string, prevWord: string): boolean` — checks every letter in prevWord has count <= count in newWord.
       - `multisetDiff(newWord: string, prevWord: string): string[]` — returns array of extra letters in newWord vs prevWord.

    2. **Exported functions**:
       - `isInDictionary(word: string, dictionary: Set<string>): boolean` — expands Q tiles, lowercases, checks `dictionary.has()`. Dictionary is passed as parameter (dependency injection per research Pattern 7).
       - `validateWord(word: string, hand: string[], dictionary: Set<string>): WordValidationResult` — checks dictionary first (reason: 'not_a_word'), then checks hand has all required letters via letterFrequency comparison (reason: 'letters_unavailable').
       - `validateTurn(newWord: string, prevWord: string, hand: string[], dictionary: Set<string>, config: ValidationConfig): TurnValidationResult` — checks in order: (1) dictionary lookup, (2) multiset superset, (3) diff length === 1, (4) added letter in hand, (5) plural ban if config.banPluralS and added letter is 'S' and newWord === prevWord + 'S'. No max word length constraint (GAME-09). Returns appropriate reason for each failure.

    Import types from `../types/game`. All functions operate on uppercase internally. The dictionary Set contains lowercase strings (from parseWordList), so lookup lowercases after Q expansion.
