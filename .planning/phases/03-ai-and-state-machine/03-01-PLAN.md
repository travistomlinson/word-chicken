  <name>Task 1: Define GameState types and implement gameReducer</name>
  <files>src/types/game.ts, src/lib/gameReducer.ts, src/lib/__tests__/gameReducer.test.ts</files>
  <behavior>
    - START_GAME creates GameState with phase='SETUP', two players (human + ai), fresh round, zeroed scores
    - SUBMIT_STARTING_WORD from SETUP validates word via validateStartingWord; on success transitions to AI_THINKING if next player is AI, HUMAN_TURN if next is human
    - SUBMIT_STARTING_WORD from non-SETUP phase returns unchanged state
    - SUBMIT_WORD from HUMAN_TURN validates via validateTurn; on success: updates currentWord, adds scoreWord(word) to player score, appends to turnHistory, draws tile back to 9, advances currentPlayerId, transitions to AI_THINKING
    - SUBMIT_WORD from AI_THINKING validates via validateTurn; on success: same updates, transitions to HUMAN_TURN
    - SUBMIT_WORD with invalid word returns unchanged state (no error thrown)
    - ELIMINATE_PLAYER removes player from activePlayers; if one player left, transitions to ROUND_END
    - END_ROUND accumulates roundScores into totalScores
    - NEXT_ROUND calls startNextRound, resets to SETUP
    - RESET_GAME returns null/initial state
    - After SUBMIT_WORD, player hand shrinks by 1 used tile then draws back to 9 from bag
    - Integration: full round from SETUP -> starting word -> human turn -> AI turn -> elimination -> ROUND_END completes without illegal transition
  </behavior>
  <action>
    1. Extend `src/types/game.ts` with new types (keep ALL existing types, add to the file):
       - `TurnPhase` type: 'SETUP' | 'HUMAN_TURN' | 'AI_THINKING' | 'ROUND_END' | 'GAME_OVER'
       - `GameConfig` interface: { difficulty: 'easy' | 'medium' | 'hard'; banPluralS: boolean; tileDistribution: TileDistribution }
       - `GameState` interface: { phase: TurnPhase; round: RoundState; config: GameConfig; roundScores: Record<string, number>; totalScores: Record<string, number> }
       - `GameAction` discriminated union: START_GAME | SUBMIT_STARTING_WORD | SUBMIT_WORD | ELIMINATE_PLAYER | END_ROUND | NEXT_ROUND | AI_TURN_START | RESET_GAME

    2. Create `src/lib/__tests__/gameReducer.test.ts` with failing tests covering all behaviors listed above. Tests should construct controlled state directly (set specific hands, bag contents) rather than relying on random deals. Inject dictionary as a small Set of known words (e.g., Set(['cat', 'cart', 'carts', 'scat', 'trace', 'crate'])).

       IMPORTANT: The reducer needs dictionary access. Since the reducer is pure, pass dictionary through GameState or through the action payload. Research suggests accessing via Zustand getState() but that couples the reducer to the store. Better approach: add `dictionary: Set<string>` to GameConfig (set once at START_GAME from the dictionarySlice). This keeps the reducer pure and testable.

    3. Create `src/lib/gameReducer.ts`:
       - `createInitialGameState(config: GameConfig): GameState` — creates two players 'human' and 'ai' via createRoundState
       - `gameReducer(state: GameState, action: GameAction): GameState` — pure reducer
       - Helper `advanceTurn(round: RoundState): string` — returns next player ID from activePlayers
       - Helper `applyValidTurn(round, playerId, word, points, config): RoundState` — updates currentWord, hand (remove used letter + drawToNine), score, turnHistory, advances currentPlayerId
       - Helper `nextPlayerIsAI(round): boolean` — checks if next currentPlayerId is 'ai'
       - For hand mutation after valid turn: compute the diff letter between newWord and currentWord (the single added letter), remove one instance of that letter from the player's hand array, then call drawToNine to refill from bag. NOTE: drawToNine mutates the bag array — this is fine because the reducer creates a new bag reference via spread before mutating.
       - For SUBMIT_STARTING_WORD: remove all letters of the starting word from the player's hand, then drawToNine. Starting words use 3 letters, so hand goes from 9 to 6, then draws back to 9.
       - Guard every action with phase check. In dev mode (import.meta.env.DEV), console.warn on illegal transitions.
       - ELIMINATE_PLAYER: call eliminatePlayer from roundManager, then checkRoundEnd. If round over, set phase to ROUND_END.
       - END_ROUND: compute roundScores from turnHistory, add to totalScores.
       - NEXT_ROUND: call startNextRound from roundManager with winner as starting player, reset phase to SETUP.

    4. Run tests: `npm test -- --run src/lib/__tests__/gameReducer.test.ts`
