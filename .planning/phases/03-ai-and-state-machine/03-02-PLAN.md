  <action>
    Create `src/hooks/useAI.ts`:

    ```typescript
    import { useEffect } from 'react'
    import { useGameStore } from '../store/gameSlice'
    import { findAIMove, findAIStartingWord, getVocabulary } from '../lib/aiEngine'
    ```

    The hook:
    1. Reads `gameState.phase` and `gameState.round.currentPlayerId` from useGameStore
    2. When `phase === 'AI_THINKING'`:
       - Schedule computation inside `requestAnimationFrame` so React paints the "thinking" state first
       - Get vocabulary via `getVocabulary(config.difficulty, config.dictionary)`
       - Shuffle the vocabulary array (Fisher-Yates or simple random sort) for move variety
       - Call `findAIMove(currentWord, aiHand, shuffledArray, vocabSet, config)`
       - If move found: dispatch `{ type: 'SUBMIT_WORD', word: move, playerId: 'ai' }`
       - If no move: dispatch `{ type: 'ELIMINATE_PLAYER', playerId: 'ai' }`
    3. When `phase === 'SETUP' && currentPlayerId === 'ai'`:
       - AI won the previous round and must submit a starting word
       - Schedule inside rAF
       - Call `findAIStartingWord(aiHand, config.dictionary)`
       - If word found: dispatch `{ type: 'SUBMIT_STARTING_WORD', word }`
       - If no word (extremely unlikely with 9 tiles and 200 starting words): log error, dispatch eliminate
    4. Cleanup: `cancelAnimationFrame(rafId)` in the useEffect cleanup function to prevent stale dispatches

    Guard: if dictionary is empty (size === 0) or gameState is null, return early from the effect.

    This hook is mounted once at the Game screen level. It is not tested with unit tests (rAF + React hooks are integration territory). Correctness of the AI logic is covered by aiEngine.test.ts.
