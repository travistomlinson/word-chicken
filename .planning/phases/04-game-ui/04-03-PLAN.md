**PlayerHand** (`src/components/PlayerHand.tsx`):

This component manages all tile interaction state locally and dispatches to the game store on submit.

State:
```typescript
const [stagedIndices, setStagedIndices] = useState<number[]>([])
const [error, setError] = useState<string | null>(null)
const [shaking, setShaking] = useState(false)
```

Read from store:
```typescript
const phase = useGameStore(s => s.gameState?.phase)
const hand = useGameStore(s => s.gameState?.round.players['human']?.hand ?? [])
const currentWord = useGameStore(s => s.gameState?.round.currentWord ?? '')
const roundNumber = useGameStore(s => s.gameState?.round.roundNumber)
const config = useGameStore(s => s.gameState?.config)
const dispatch = useGameStore(s => s.dispatch)
```

Reset staged tiles on phase/round change (pitfall 4 from RESEARCH):
```typescript
useEffect(() => {
  setStagedIndices([])
  setError(null)
}, [roundNumber, phase])
```

Tile interaction:
- `handleTileClick(handIndex: number)`: If index is in `stagedIndices`, remove it (unstage). Otherwise, add it (stage). This uses INDEX tracking to handle duplicate letters correctly (pitfall 6 from RESEARCH).
- `handleUnstage(stagingIndex: number)`: Remove the staging-index-th entry from `stagedIndices`.
- Derived values (computed during render, NOT in useEffect):
  - `stagedLetters = stagedIndices.map(i => hand[i])` -- the letters currently staged
  - `remainingHand` -- hand tiles NOT in stagedIndices, preserving order with their original indices

Submit logic:
- `handleSubmit()`:
  - Build word from staged letters: `const word = stagedLetters.join('')`
  - If `phase === 'SETUP'`:
    - Pre-validate: `validateStartingWord(word, hand, config.dictionary)` -- import from `../lib/roundManager`
    - If invalid: map reason to user-friendly text:
      - `'not_a_word'` -> "Not a valid word"
      - `'not_in_corpus'` -> "Not a valid starting word (use a common 3-letter word)"
      - `'letters_unavailable'` -> "You don't have those letters"
    - Set error, trigger shake (set `shaking = true`, `setTimeout(() => setShaking(false), 500)`)
    - Return without dispatching
    - If valid: `dispatch({ type: 'SUBMIT_STARTING_WORD', playerId: 'human', word })`
  - If `phase === 'HUMAN_TURN'`:
    - Pre-validate: `validateTurn(word, currentWord, hand, config.dictionary, { banPluralS: config.banPluralS })`
    - If invalid: map reason to user-friendly text:
      - `'not_a_word'` -> "Not a valid word"
      - `'letter_not_in_hand'` -> "You don't have those letters"
      - `'not_a_superset'` -> "Must use all letters from the current word plus one new letter"
      - `'plural_banned'` -> "Adding S to pluralize is not allowed"
    - Set error, trigger shake, return
    - If valid: `dispatch({ type: 'SUBMIT_WORD', playerId: 'human', word })`

Render:
- If `phase === 'AI_THINKING'`: render AI thinking animation (see below) instead of the normal hand
- If `phase === 'ROUND_END'` or `phase === 'GAME_OVER'`: render nothing (hand hidden during results)
- Normal render (SETUP or HUMAN_TURN):
  - `StagingArea` at top with stagedLetters, handlers, error, shaking state
  - Hand tiles below: `flex gap-1 sm:gap-2 flex-wrap justify-center mt-4`
    - Each tile in `remainingHand` (tiles NOT staged): `TileCard` with `color="concrete"` `size="md"` `onClick={() => handleTileClick(originalIndex)}`
    - Staged tiles shown with reduced opacity or hidden from hand (since they're in staging area)

**AI Thinking Animation** (rendered inside PlayerHand when `phase === 'AI_THINKING'`):

Implementation:
```typescript
const LETTERS = 'ABCDEFGHIJKLMNOPRSTUW'
const [displayLetters, setDisplayLetters] = useState<string[]>(['?','?','?','?'])

useEffect(() => {
  if (phase !== 'AI_THINKING') return
  // Show 3-5 placeholder tiles cycling through random letters
  const tileCount = Math.max(3, (currentWord.length + 1))  // expected word length
  setDisplayLetters(Array(Math.min(tileCount, 8)).fill('?'))

  const interval = setInterval(() => {
    setDisplayLetters(prev => prev.map(() =>
      LETTERS[Math.floor(Math.random() * LETTERS.length)]
    ))
  }, 80)
  return () => clearInterval(interval)
}, [phase, currentWord.length])
```

Render the cycling letters as `TileCard` components with `color="red"` and `disabled`, inside a container with `flex gap-1 justify-center`.

Also show the human player's hand tiles below (disabled, `opacity-50`) so the player can still see what they have.
